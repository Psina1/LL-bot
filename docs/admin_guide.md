# Инструкция администратора Telegram-бота

Этот документ для админов, которые работают с ботом через Telegram и не имеют доступа к серверу.

## Кто считается админом

Админ определяется по `TELEGRAM_ADMIN_IDS` в `.env`.

Если пользователь не в списке admin IDs, команды админки ему недоступны.

## Как открыть админку

В Telegram отправить:

```text
/admin
```

Появятся кнопки:
- `Админ: статус`
- `Админ: загрузить материал`
- `Админ: материалы`
- `Админ: тексты`
- `Админ: статистика`
- `Главное меню`

Опасных кнопок удаления материалов или сброса базы нет.

## Админ: статус

Кнопка проверяет:
- соединение с базой;
- количество документов, чанков и вопросов;
- LLM chat;
- embeddings;
- последние ошибки;
- режим запуска;
- лимит размера файла.

Нормально:

```text
База данных: OK
LLM chat: OK
Embeddings: OK
```

Если есть ошибка:
- не паниковать;
- сделать скрин;
- отправить разработчику;
- не нажимать `/reindex` без причины.

## Админ: загрузить материал

Это мастер загрузки общего материала программы.

Шаги:
- выбрать сезон;
- выбрать модуль;
- выбрать тип материала;
- отправить файл.

Поддерживаемые форматы:
- PDF;
- DOCX;
- PPTX;
- TXT.

Лимит размера задаётся в `.env`, сейчас обычно 20 MB.

## Как выбирать тип материала

`Тип: материалы занятия`

Использовать для:
- презентаций;
- методичек;
- конспектов;
- файлов занятия.

`Тип: домашнее задание`

Использовать для:
- файлов с ДЗ;
- инструкций по выполнению;
- критериев сдачи.

`Тип: саммари`

Использовать для:
- готовых выжимок;
- кратких резюме занятий.

`Тип: расписание`

Использовать для:
- программы обучения;
- расписания занятий;
- организационных дат.

`Тип: другое`

Использовать только если материал не подходит к другим типам.

## После загрузки материала

Бот должен ответить:

```text
Материал загружен и обработан.
```

Если бот пишет, что файл не обработался:
- проверить формат файла;
- проверить размер файла;
- попробовать другой файл;
- открыть `Админ: статус`;
- отправить разработчику текст ошибки или скрин.

## Админ: материалы

Показывает последние материалы:
- id;
- название;
- видимость;
- модуль;
- статус.

Статусы:
- `ready` - материал готов;
- `processing` - материал обрабатывается;
- `error` - была ошибка обработки;
- `uploaded` - материал создан, но ещё не готов.

## Админ: тексты

Можно изменить:
- приветствие;
- помощь;
- расписание.

Как работает:
- нажать `Админ: тексты`;
- выбрать нужный текст;
- отправить новый текст одним сообщением;
- проверить предпросмотр;
- нажать `Сохранить текст` или `Отменить`.

Новый текст начинает использоваться сразу.

## Рекомендации по текстам

Писать коротко.

Не вставлять:
- секреты;
- API-ключи;
- внутренние ссылки с ограниченным доступом;
- персональные данные участников.

Если в тексте есть дата, лучше писать явно:

```text
21 мая 2026, 11:00
```

## Админ: статистика

Показывает:
- количество пользователей;
- количество документов;
- количество чанков;
- количество вопросов;
- полезные ответы;
- неполезные ответы;
- причины `Не полезно`;
- последние ошибки.

Причины `Не полезно`:
- `Не нашёл ответ`;
- `Слишком общий`;
- `Не понял вопрос`;
- `Другая причина`.

## /reindex

Команда существует, но не вынесена кнопкой в меню.

Она переиндексирует документы и может:
- занять время;
- потратить embeddings;
- нагрузить OpenAI API.

Использовать только если:
- материалы загружены, но бот их не видит;
- разработчик попросил выполнить команду;
- после крупной правки индексации нужно обновить chunks.

## Что админам нельзя делать

- Не просить токены у разработчика.
- Не отправлять `.env` в чат.
- Не загружать персональные данные без согласования.
- Не загружать огромные файлы пачками.
- Не использовать `/reindex` просто “проверить”.
- Не пытаться удалить данные через сервер, если нет доступа и инструкции.

## Быстрый сценарий перед занятием

1. Нажать `/admin`.
2. Нажать `Админ: статус`.
3. Проверить OK по БД, LLM и embeddings.
4. Нажать `Админ: материалы`.
5. Убедиться, что нужные материалы есть.
6. В обычном меню задать тестовый вопрос по материалам.
7. Если ответ плохой, нажать `Не полезно` и указать причину.
