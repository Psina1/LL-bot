# Видео в Telegram: заготовка для MVP

## Цель

Подготовить бота к просмотру записей занятий внутри Telegram или через ссылку, не ломая текущий MVP.

Сейчас видео-ветка скрыта:

```env
VIDEO_LIBRARY_ENABLED=false
VIDEO_LIBRARY_URL=
```

## Где будет находиться кнопка

Планируемое место:

```text
Материалы программы
-> Сезон 1. Бизнес-консалтинг
-> Видео занятий
```

Пока `VIDEO_LIBRARY_ENABLED=false`, кнопка не показывается пользователям.

## Варианты реализации

### 1. Ссылка на корпоративную видеотеку

Бот отправляет кнопку со ссылкой на внешний сервис.

Плюсы:
- быстрее всего подключить;
- видео остаются в корпоративном хранилище;
- не нужно хранить тяжёлые файлы на VM.

Риски:
- если сервис работает только через корпоративный VPN, телефон пользователя тоже должен иметь доступ;
- ссылка может не открыться внутри Telegram WebView;
- нужно понять, можно ли шарить такие ссылки участникам.

### 2. Telegram Mini App

Бот открывает Web App внутри Telegram.

Плюсы:
- выглядит как встроенный раздел бота;
- можно сделать каталог занятий, фильтры и описания;
- позже можно добавить прогресс просмотра.

Риски:
- нужен HTTPS web-интерфейс;
- если видеотека доступна только через VPN, Mini App тоже должен иметь сетевой доступ;
- нужно проверить, разрешает ли видеосервис встраивание в iframe/WebView;
- это уже отдельный маленький frontend, не часть первого MVP.

### 3. Отправка видео прямо в Telegram

Бот отправляет видеофайл или документ в чат.

Плюсы:
- пользователю удобно;
- не нужен внешний видеоплеер.

Риски:
- лимиты Telegram на размер файлов;
- тяжёлые файлы быстро съедят диск/трафик;
- возможны ограничения по распространению записей.

## Что нужно узнать у владельцев видео

- Где лежат записи занятий?
- Есть ли стабильные ссылки на конкретные занятия?
- Работают ли ссылки с телефона вне корпоративной сети?
- Нужен ли VPN на устройстве пользователя?
- Можно ли открывать видео внутри Telegram WebView?
- Можно ли встраивать видео в iframe?
- Нужна ли авторизация пользователя?
- Можно ли показывать записи всем участникам программы?
- Есть ли ограничения по скачиванию/пересылке?

## Как включить заготовленную ветку

В `.env` на сервере:

```env
VIDEO_LIBRARY_ENABLED=true
VIDEO_LIBRARY_URL=https://example.internal/video-library
VIDEO_LIBRARY_TITLE=Видео занятий
VIDEO_ACCESS_NOTE=Видео открываются через корпоративную видеотеку. Если ссылка не открывается, проверь VPN.
```

После изменения `.env`:

```bash
cd ~/apps/LL-bot
docker compose up -d
```

## Важное ограничение

Если видеотека работает только под корпоративным VPN, Telegram сам по себе этот VPN не заменяет.

Нужно будет проверить реальный сценарий:
- телефон пользователя с VPN;
- Telegram Desktop на рабочем ноутбуке под VPN;
- доступность ссылки внутри Telegram WebView;
- доступность обычной ссылки в браузере.
