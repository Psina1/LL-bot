# Карта кнопок Telegram-бота

## Старт

### `/start`
Показывает приветственный текст и второе сообщение с выбором времени уведомлений.

Inline-кнопки:
- `09:00`
- `12:00`
- `15:00`

Переход:
- Главное меню

### `/help` или `Помощь`
Показывает инструкцию по использованию бота.

Переход:
- Главное меню

### `/cancel`
Сбрасывает текущий сценарий.

Переход:
- Главное меню

## Главное меню

Кнопки:
- `Задать вопрос`
- `Материалы программы`
- `Домашние задания`
- `Расписание Лиги Лидеров`
- `Настройки уведомлений`

Скрыто фичефлагом `SHOW_PROJECT_CONTEXT_MENU=false`:
- `Уточнить контекст моего проекта`

Скрыто до согласования сценария:
- `Нужна помощь с проектом`

## Задать вопрос

Кнопка:
- `Задать вопрос`

Что происходит:
- Бот пишет: `Выбери раздел по своему вопросу:`
- Пользователь выбирает inline-кнопку: `Вопрос по программе`, `Технический вопрос`, `Другое`
- Бот пишет: `Напиши свой вопрос.`
- Пользователь пишет вопрос
- Бот учитывает выбранный раздел
- Для технических вопросов бот подсказывает писать в Telegram: `@reptiloid0`
- Для вопросов по программе бот ищет релевантный контекст
- Бот отправляет вопрос и контекст в LLM
- Бот сохраняет вопрос и ответ в БД

Переход после ответа:
- Главное меню

## Материалы программы

Кнопка:
- `Материалы программы`

Что происходит:
- Бот показывает inline-кнопки разделов материалов прямо в сообщении

Inline-кнопки:
- `Записи и материалы занятий`
- `Подкасты на основе занятий`
- `Саммари занятий`
- `Главное меню`

Старый текстовый путь через `Сезон 1. Бизнес-консалтинг` оставлен в коде для совместимости, но в основном меню больше не показывается.

### Записи и материалы занятий

Что происходит:
- Если видео загружены, бот показывает список записей inline-кнопками
- Пользователь выбирает конкретную запись, и бот отправляет только её
- Отдельная inline-кнопка `Текстовые материалы` показывает список RAG-документов
- В списке есть `id` каждого материала
- Если видео нет, бот сразу показывает текстовые материалы

Как спросить по конкретному файлу:
- написать `материал 1: кратко что в этом файле?`
- вместо `1` использовать id нужного материала из списка

Переход:
- Главное меню

### Подкасты на основе занятий

Что происходит:
- Если админ загрузил аудиоподкасты, бот показывает список подкастов inline-кнопками
- Пользователь выбирает конкретный подкаст, и бот отправляет только его
- Если аудиоподкастов нет, бот предлагает кнопку `Сделать текстовую подкаст-выжимку`

MVP-пайплайн подкастов:
- админ загружает готовый аудиофайл через `Админ: загрузить медиа`
- бот сохраняет Telegram `file_id`
- пользователь выбирает подкаст из списка
- бот отправляет аудиофайл

Переход после ответа:
- Главное меню

### Саммари занятий

Что происходит:
- Бот просит LLM сделать краткое саммари по загруженным материалам сезона
- Если материалов нет, бот пишет, что не нашёл точного ответа

Переход после ответа:
- Главное меню

## Домашние задания

Кнопка:
- `Домашние задания`

Что происходит:
- Бот показывает подменю домашек

Подменю:
- `Список заданий`
- `Помощь с домашкой`
- `Главное меню`

### Список заданий

Что происходит:
- Бот ищет документы с `material_type=homework`
- Также ищет материалы, похожие на домашние задания по названию
- Если ничего не найдено, честно пишет об этом

Переход:
- Главное меню

### Помощь с домашкой

Что происходит:
- Бот пишет: `Задай вопрос по заданию.`
- Пользователь задаёт вопрос
- Бот отвечает через LLM с учётом материалов и проектного контекста

Переход после ответа:
- Главное меню

## Уточнить контекст моего проекта

Статус:
- временно скрыто из главного меню
- код и обработчики оставлены для будущего возврата
- включение через `SHOW_PROJECT_CONTEXT_MENU=true`

Кнопка:
- `Уточнить контекст моего проекта`

Что происходит:
- Бот просит загрузить контекст департамента или проекта
- Если текстовый контекст уже сохранён, бот показывает краткий фрагмент

Подменю:
- `Загрузить файл с контекстом`
- `Добавить контекст текстом`
- `Главное меню`

### Загрузить файл с контекстом

Что происходит:
- Бот просит прислать PDF/DOCX/PPTX/TXT
- Пользователь отправляет файл
- Бот сохраняет файл в личный контекст пользователя
- Бот извлекает текст
- Бот режет текст на чанки
- Бот создаёт embeddings
- Бот сохраняет chunks в БД
- Бот пишет: `Спасибо! Теперь я знаком с этим контекстом и материалами`

Переход после успешной загрузки:
- Главное меню

### Добавить контекст текстом

Что происходит:
- Бот просит описать проект в 3-5 предложениях
- Пользователь пишет описание
- Бот сохраняет текст в `users.project_context`

Переход:
- Подменю контекста проекта

## Настройки уведомлений

Кнопка:
- `Настройки уведомлений`

Что происходит:
- бот показывает текущую настройку времени
- пользователь выбирает одно время для всех уведомлений

Подменю:
- `Уведомления: 09:00`
- `Уведомления: 12:00`
- `Уведомления: 15:00`
- `Уведомления: отключить`
- `Главное меню`

Что отправляется сейчас:
- тестовый текст уведомления
- `.ics` файл с короткой подсказкой для календаря, если админ его загрузил

Важно:
- настройка времени одна для всех направлений
- позже по этому же времени можно отправлять несколько уведомлений: оргмоменты, домашние задания, занятия, дедлайны

Переход:
- Главное меню

## Свободный текст

Если пользователь пишет текст без выбора сценария:
- Если текст начинается с `материал N:`, бот отвечает только по материалу с этим id
- Бот воспринимает сообщение как обычный вопрос
- Бот ищет релевантный контекст
- Бот отвечает через LLM
- Бот сохраняет вопрос и ответ

Переход после ответа:
- Главное меню

## Админские команды

Команды доступны только `TELEGRAM_ADMIN_IDS`.

### `/admin`
Показывает безопасное админское меню.

Кнопки:
- `Админ: статус`
- `Админ: загрузить материал`
- `Админ: материалы`
- `Админ: тексты`
- `Админ: статистика`
- `Админ: напоминание`
- `Админ: загрузить ICS`
- `Админ: загрузить медиа`
- `Админ: расходы`
- `Главное меню`

Опасных кнопок удаления материалов или сброса базы нет.

### `Админ: статус`

Что происходит:
- Проверяет соединение с БД
- Показывает количество документов, чанков и вопросов
- Проверяет LLM chat
- Проверяет embeddings
- Показывает последние ошибки
- Показывает режим запуска и лимит файла

### `Админ: загрузить материал`

### `/upload_global_material`
Запускает мастер загрузки общего материала.

Шаги:
- выбрать сезон: `Сезон 1. Бизнес-консалтинг` или `без сезона`
- выбрать привязку: `общий материал` или `урок/модуль 1-4`
- выбрать тип: материалы занятия, домашнее задание, саммари, расписание, другое
- отправить файл PDF/DOCX/PPTX/TXT

После обработки материал становится общим для всех пользователей.
Бот сохраняет служебные ключи и теги: `lesson_key`, `lesson`, `module`, `type`, `scope`.

Старый caption-формат всё ещё поддерживается технически, но обычному админу его помнить не нужно.

### `Админ: загрузить медиа`

Запускает мастер загрузки видео или подкаста.

Шаги:
- выбрать тип: `Медиа: видео` или `Медиа: подкаст`
- выбрать привязку: `общий материал` или `урок/модуль 1-4`
- отправить видео или аудиофайл

Бот сохраняет Telegram `file_id`, а не сам большой файл.
Дополнительно сохраняются `lesson_key` и теги вроде `media:video`, `lesson:1`, `scope:general`.

Где показывается:
- видео: `Материалы программы` -> `Записи и материалы занятий`
- подкасты: `Материалы программы` -> `Подкасты на основе занятий`

### `Админ: материалы`

### `/list_materials`
Показывает список документов.

### `/reindex`
Переиндексирует документы.

Команда оставлена slash-командой, чтобы случайно не нажать её в меню. Она не удаляет документы, но может занять время и потратить embeddings.

### `Админ: статистика`

### `/stats`
Показывает базовую статистику:
- пользователи
- пользователи с проектным контекстом
- документы
- общие материалы и личные файлы
- статусы документов
- чанки
- вопросы всего и за периоды
- размер файлов
- ошибки
- полезные/неполезные ответы
- причины `Не полезно`
- последние ошибки

### `Админ: расходы`

Показывает ориентировочные расходы:
- VM/сервер в час, день и месяц
- примерный расход VM с даты `VM_BILLING_STARTED_AT`, если она задана
- LLM-токены за 24 часа, 7 дней и 30 дней
- оценку OpenAI в рублях
- оценку YandexGPT Lite в рублях
- оценку embeddings по загруженным чанкам

Точные суммы нужно сверять в Yandex Billing и кабинете LLM-провайдера.

### `Админ: напоминание`

Позволяет отправить сообщение в групповой чат программы.

Что происходит:
- бот просит текст напоминания
- админ отправляет текст одним сообщением
- бот публикует текст в привязанный групповой чат

Если чат ещё не привязан:
- бот подсказывает добавить его в будущий чат программы
- админ отправляет в групповом чате команду `/set_group_chat`

Дополнительные команды:
- `/set_group_chat` - привязать текущий групповой чат для напоминаний
- `/group_chat_status` - проверить привязанный чат
- `/send_reminder текст` - отправить напоминание без кнопки

Обычные сообщения участников в групповом чате бот игнорирует. Ответы на вопросы работают в личном чате с ботом.

### `Админ: тексты`

Позволяет изменить тексты без доступа к серверу:
- `Изменить приветствие`
- `Изменить помощь`
- `Изменить расписание`
- `Изменить текст уведомления`

Как работает:
- Админ выбирает текст
- Бот показывает текущую версию
- Админ отправляет новый текст одним сообщением
- Бот показывает предпросмотр
- Админ нажимает `Сохранить текст` или `Отменить`
- Новый текст сразу используется в `/start`, `Помощь`, `Расписание Лиги Лидеров` или уведомлениях
- Встроенные Telegram-гиперссылки сохраняются и отправляются как ссылки

### `Админ: загрузить ICS`

Позволяет загрузить календарный файл для уведомлений.

Что происходит:
- админ нажимает `Админ: загрузить ICS`
- бот просит прислать `.ics`
- админ отправляет файл
- бот сохраняет файл в `data/notifications`
- при отправке уведомлений бот прикладывает этот файл пользователю

Удаления текстов нет. Если нужен возврат к дефолтному тексту, пока проще отправить старый текст заново.

## Оценка ответа

После LLM-ответа бот показывает inline-кнопки:
- `Полезно`
- `Не полезно`

Что происходит:
- Оценка сохраняется в `message_feedback`
- Если пользователь нажал `Не полезно`, бот уточняет причину
- Админ видит суммарные оценки в статистике
- Это помогает понять, где бот отвечает плохо

Причины `Не полезно`:
- `Не нашёл ответ`
- `Слишком общий`
- `Не понял вопрос`
- `Другая причина`

## Убрано из пользовательского меню

Эти кнопки больше не показываются:
- `Подробнее`
- `Короче`
- `Сделать чек-лист`
- `Задать уточнение`
- `Найти риски`
- `Сформировать вопросы`
- `Применить инструменты обучения к проекту`
- `Сформировать список рисков`
- `Сформировать вопросы к стейкхолдерам`

Идея:
- Пользователь может написать эти просьбы обычным текстом в чат
- Меню остаётся коротким и не перегружает участника
